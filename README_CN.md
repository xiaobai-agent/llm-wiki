# 🧠 LLM Wiki

**开源的个人知识库，让你的 AI 越用越聪明。Schema 驱动，支持任何大模型（ChatGPT、Claude、Gemini），自我进化。60 秒上手。**

> 灵感来自 [Andrej Karpathy 的 llm-wiki](https://gist.github.com/karpathy/1dd0294ef9567971c1e4348a90d69285)。不只是一个理念，而是一个**开箱即用的 starter kit**。不需要装 App，没有订阅费——只有 Markdown 文件和你喜欢的 AI。

## 💡 核心理念：让知识产生复利

大多数人把 AI 当**无状态工具**——每次对话都从零开始。

LLM Wiki 把你的 AI 变成一个**学习系统**。你喂入的知识越多，它就越聪明——不是通过训练，而是通过结构化的知识积累。

看看会发生什么：

```
第 1 周：  存入 5 篇文章 → AI 整理成概念页面
第 4 周：  30 篇素材 → AI 开始发现你自己都没注意到的关联
第 12 周：100+ 篇素材 → AI 发现跨领域的规律和洞察
          → 这些发现反过来优化 AI 的策略和技能
          → 你的 AI 在进化。不是因为你重新训练了它，
             而是因为它的知识在产生复利。
```

**这就是 AI Agent 自我进化的基础。** 不是微调，不是每次查询都重新检索的 RAG。而是一个活的、会生长的知识库，每一条新信息都让其他所有信息变得更有价值。

Karpathy 管这叫"编译知识"。我们造了这台编译器。

## ⚡ 快速开始（60 秒）

### 第一步：Clone

```bash
git clone https://github.com/xiaobai-agent/llm-wiki.git
cd llm-wiki
```

### 第二步：粘贴 Schema

复制 [`WIKI-SCHEMA.md`](WIKI-SCHEMA.md) 的全部内容，粘贴到你的 AI 助手中（Claude、ChatGPT、Gemini 或任何大模型）。

跟它说：

> "这是我的知识库 Schema。请按照这个规范帮我维护一个个人 Wiki。我会给你发文章、笔记和想法，你来整理。"

### 第三步：开始添加知识

给你的 AI 发送任何内容：

- 📰 "这篇关于 X 的文章，帮我存进知识库"
- 💡 "我刚学到了 Y，记一下"
- 🔗 "总结一下这个链接并存入：https://..."

**搞定。你现在拥有了一个会随你成长的个人知识库。**

## 🔄 自我进化循环

这是 LLM Wiki 与笔记软件的本质区别：

```
                    ┌──────────────────┐
          ┌────────▶│   原始知识        │────────┐
          │         │ （文章、笔记、     │        │
          │         │  视频、想法）      │        ▼
          │         └──────────────────┘  ┌─────────────┐
          │                               │  AI 整理     │
          │                               │  & 关联      │
          │                               └──────┬──────┘
          │                                      │
          │                                      ▼
   ┌──────┴──────┐                      ┌──────────────┐
   │  更好的      │◀─────────────────────│  Wiki 页面    │
   │  策略与技能  │     发现反哺          │ （概念、实体、│
   │             │                      │  洞察）       │
   └─────────────┘                      └──────────────┘
```

**阶段一：知识积累**
AI 把原始内容整理成结构化页面。每加入一篇新素材，概念页就更丰富一分。

**阶段二：自我发现**
知识积累到一定量，AI 开始发现规律——概念之间的关联、来源之间的矛盾、整体大于部分之和的洞察。这些被捕获为**洞察页（insight pages）**。

**阶段三：策略进化**
发现反过来优化策略。AI 学会了*怎么做研究更高效*、*哪些来源更可靠*、*该关注什么规律*。这些记录在**元知识（meta/）**——AI 自己的操作手册，会随时间自我改进。

**结果：一个越来越擅长自己工作的 AI，不是因为你告诉它怎么做，而是因为它的知识库教会了它。**

### 🔍 具体案例：洞察是怎么涌现的

假设你几周内陆续存入了这四篇素材：

| 素材 | 领域 | 关键事实 |
|------|------|---------|
| 政府政策报告 | 法规 | 全国仅12家企业获得定制化妆品试点许可 |
| 某公司调研 | 商业 | A 公司有 GMP 产线但缺 AI 能力 |
| 你自己的业务数据 | 数据 | 你有5万客户、10年追踪数据 |
| 设备厂商调研 | 技术 | 机械臂调配 MVP 成本仅约 ¥7万 |

每篇素材各自待在自己的页面里。有用，但孤立。

然后某天你问 AI：*"跟 A 公司合作的最佳路径是什么？"*

AI 交叉引用四篇素材，发现了**任何单篇都没说过的东西**：

> 💡 **洞察：** "你出 AI + 设备，A 公司出牌照 + 场地。直接入股最优，因为不影响试点许可证——而你10年的追踪数据集是独一无二的谈判筹码，没有竞争对手能复制。"

这个结论不存在于任何单篇素材中。它是从法规 + 商业 + 数据 + 技术的交叉引用中**涌现**出来的。

**这就是被捕获的洞察。** AI 把它保存为洞察页，引用全部四个来源，从此这个战略分析永久可用——不需要每次都从头推导。

**这就是"自我进化"在实践中的意思。** 不是抽象的 AI 魔法。是实实在在的跨领域发现，让你的决策能力产生复利。

> *"RAG 每次查询都重新发现知识。Wiki 编译一次，复利永远。"*
> — Karpathy llm-wiki 的核心洞见

## 🤔 为什么需要 LLM Wiki？

**问题：** 你每天读大量文章、看无数视频、灵光一闪有绝妙的想法——然后一周之内忘掉 90%。

**传统方案都不好用：**
- 收藏夹 → 未读链接的坟场
- 笔记软件 → 有组织的拖延症
- 稍后阅读 → 永远不读
- RAG → 每次查询都重新检索，没有积累

**LLM Wiki 不一样：**
- **AI 帮你整理。** 你只管往里扔内容
- **知识会复利增长。** 新信息自动与已有知识关联
- **洞察会自动涌现。** 交叉引用揭示你手动永远发现不了的关联
- **策略会自我进化。** 系统学会如何更好地学习
- **Schema 而非软件。** 不需要装 App，没有订阅费，没有锁定。只有 Markdown 文件和你喜欢的 AI

## 📁 工作原理

### 三层架构

```
wiki/
├── raw/                ← 第一层：原始素材（只读）
│   ├── articles/       ← 网页文章、博客
│   ├── notes/          ← 你的个人笔记和想法
│   ├── videos/         ← 视频转录稿
│   └── other/          ← PDF、文档等
│
├── pages/              ← 第二层：知识页面（AI 维护）
│   ├── concepts/       ← 主题概览（如"机器学习.md"）
│   ├── entities/       ← 人物、公司、产品（如"openai.md"）
│   ├── sources/        ← 单篇素材摘要
│   ├── comparisons/    ← 横向对比分析
│   └── insights/       ← 跨素材发现的洞察 ✨
│
├── meta/               ← 第三层：策略记忆（AI 自己的操作手册）
│   ├── ingest-strategies.md    ← 不同素材类型的最佳实践
│   ├── research-patterns.md    ← 高效研究模板
│   ├── source-quality.md       ← 来源可信度评级
│   └── failure-log.md          ← 踩坑教训
│
└── index.md            ← 全页面目录索引
```

**第一层（Raw）** = 你的原始素材，不动。这是 ground truth。

**第二层（Pages）** = AI 策展的知识页面，随时间越来越丰富。自我发现在这里发生——洞察从多源交叉引用中涌现。

**第三层（Meta）** = AI 自己的策略记忆。怎么做研究更高效，哪些来源值得信赖，踩过什么坑。自我进化在这里发生。

**索引** = 你的目录。始终保持最新。

## 📋 Schema 概览

[`WIKI-SCHEMA.md`](WIKI-SCHEMA.md) 是你知识库的大脑。它告诉 AI：

- **怎么组织** — 目录结构、文件命名、页面类型
- **怎么格式化** — YAML frontmatter、Markdown 规范、交叉引用
- **怎么入库** — 添加新内容的分步工作流
- **怎么查询** — 从知识库搜索和回答问题
- **怎么进化** — 什么时候捕获洞察、什么时候更新策略
- **怎么维护** — 自检程序，保持质量

你可以自定义每一个部分。这是**你的**知识库。

## 🧰 工具集

[`tools/`](tools/) 目录包含独立的实用工具：

### [ASCII 渲染器](tools/ascii-renderer/) — 字符画转 PNG

将 ASCII 字符画（架构图、流程图、时间线）渲染为高清 PNG 图片，代码编辑器风格。

```bash
npm install puppeteer
node tools/ascii-renderer/render.js diagram.txt output.png --title "系统架构"
```

支持亮色/暗色主题、行号、Retina 分辨率、Unicode + Emoji。

### [Schema 验证器](tools/schema-validator/) — 知识库体检

零依赖，检查目录结构、frontmatter 格式、交叉引用、孤立页面。

```bash
node tools/schema-validator/validate.js ./wiki
```

支持 `--json`（CI 集成）和 `--fix`（自动修复）。
